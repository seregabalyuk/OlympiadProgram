#include <iostream>

Основной код пишется в **main**

int main() {

Чтобы написать комментарий, нужно в начале поставить `//` комментарий или `/*` комментарий `*/`.

// комментарий, тут код не выполняется

*/

# Команды

## Создать новую переменную (*struct/class*)

Переменные хранят информацию, с помощью команд мы будем изменять эту информацию. Но для начала нужно научиться создавать переменные.

Синтаксис
```
тип_переменной название;
```
Название может состоять только из букв (больших и маленьких), цифр и нижнего подчёркивания **`_`**.
Первой буквой не должна быть цифра.
Также не должно быть в одном `теле` двух одинаковых переменных.

Неподходящие названия: `1value`, `hurg.hog`

Пример:
```c++
int _value;
```

Можно сразу задать значение переменной
```
тип_переменной название(значение);
тип_переменной название = значение;
```

Пример:
```c++
int value1(0);
int value2 = 1;
```

Ещё можно создать сразу несколько переменных одного типа через запятую

Пример:
```c++
int a(0), b = 10, c = 9;
```


## Основные типы переменных

1. `int`
   - целочисленная
   - находится в дианозоне [`-2^31` : `2^31`), примерно [`-10^9`: `10^9`]
   - занимает `4 байта`
   - [Подробнее][1]
   - Пример
        ```c++
        int b = 12;
        ```
1. `long long`
   - целочисленная
   - находится в дианозоне [`-2^63` : `2^63`), примерно [`-10^18`: `10^18`]
   - занимает `8 байт`
   - [Подробнее][1]
   - Пример
        ```c++
        long long b = 12LL;
        // LL в конце числа значит, что значение в long long
        // по умолчанию в int
        ```
1. `char`
    - символ
    - занимает `1 байт`
    - может использоваться как целочисленная переменная в дианозоне [`-128` : `128`)
    - чтобы задать нужно написать `'`*символ*`'`
    - Пример 
        ```c++
        char c = 'a';
        ```
1. `unsigned` + `тип`
   - делает тип положительным
   - размер не меняет
   - для `unsigned int` диапозон [`0` : `2^32`)
   - для `unsigned long long` диапозон [`0` : `2^64`)
   - для `unsigned char` диапозон [`0` : `256`)
   - [Подробнее][1]
   - Пример
        ```c++
        unsigned int a;
        unsigned long long b = 12ULL;
        // ULL в конце числа значит, что значение в unsigned long long
        unsigned char c;
        ``` 
1. `bool`
   - принимает значения только `true`/`false`
   - занимает `1 байт`
   - может использоваться как целочисленная переменная со значениями `0` и `1` (`true = 1` и `false = 0`)
   - Пример
        ```c++
        bool b = true;
        ```
1. `float`
   - тип с плавающей точкой, то есть может хранить нецелые числа
   - занимает `4 байта`
   - [Подробнее][2]
   - Пример
        ```c++
        float a = 1.5f;
        // f в конце числа значит, что значение в float
         // по умолчанию в double
        ```
1. `double`
   - тип с плавающей точкой
   - занимает `8 байта`
   - более точный чем `float`
   - [Подробнее][2]
   - Пример
        ```c++
        double a = 1.5;
        ```
1. `long double`
   - тип с плавающей точкой
   - занимает `16 байта`
   - более точный чем `double`, самый точный
   - [Подробнее][2]
   - Пример
        ```c++
        long double a = 1.5;
        ```

Эти типы можно преобразовывать в друг друга. 

Пример:
```c++
    int a = 10;
    float b = a;
    // b = 10.0f
    int c = 99;
    unsigned int d = c;
    // d = 99
```

Исключение: `bool` нельзя перевести в `тип с плавающей точкой` и наоборот

Также `bool` будет `false` только если другой тип был равен `0`.

Пример:
```c++
    int a = -1;
    bool b = a;
    // b = true
    long long c = 1000000LL;
    bool d = c;
    // d = true
    unsigned long long e = 0;
    bool f = e
    // f = false
```

## Oбработка переменных
Обрабатывать перемменые мы можем с помощью `функций`.
Среди функций есть особые, которые называются `операторы`.

Как работает функция:

```
название_функции(тип1, тип1, ...) -> возращаемый_тип
```

> **Замечание:** Это схематичное обозначение. В с++ такого нет.

Пример функции максимума:

```
max(int, int) -> int
```

Как вызвать функцию:

```c++
int a = 10, b = 20;
int c = max(a, b);
// c = 20 
```

## Операторы
Подробно о них можно прочитать в [wikipedia][3].
Здесь собрана основаная информация.

1. ### operator `=` :
    - #### Описание:
        ```
        =(type, type) -> &type
        ```
        такой тип операторов называют `бинарным`
    - #### Как работает:
        приравнивает одно значение другому
        
        Пример:
        ```c++
        int a = 10;
        int b = 20;
        b = a;
        // b = 10
        ```
2. ### operators `+ - * /` :
    - #### Описание:
        Бинарные операторы.
    - #### Как работает:
        Выполняют стандартные арефметические операции над `базовыми типами`.
        Заметим, что при делении целочисленных переменных происходит округление.
        Для положительных округление вниз.
        Для отрицательных вверх.

        Пример:
        ```c++
        int a = 10;
        int b = 3;
        int c = a / b;
        // c = 3
        ```
3. ### operator `%` :
    - #### Описание:
        Бинарный оператор.
        Работает только с целыми типами.
    - #### Как работает: 
        Берёт левое число по модулю правого.

        Пример:
        ```c++
        int a = 10;
        int b = 3;
        int c = a % b;
        // c = 1
        ```
        Для отрицательных значений работает странно.
        Можете сами проверить.
5. ### operators `< == > != <= >=` :
    - #### Описание:
        Бинарные операторы. Но их схема отличается:
        ```
        <(type1, typ2) -> bool
        ```
        Это операторы сравнения.
    - #### Как работает:
        Сравнивают 2 числа. 
        Возращает `true` если сравнение верное и `false` иначе.

        Пример:
        ```c++
        int a = 2;
        long long b = 3;
        bool c = a < b;
        // c = true
        c = a >= b;
        // c = false 
        ``` 

        > **Замечание 1:** не путайте операторы **`=`** и **`==`**, первый приравнивает, второй сравнивает.
        
        >**Замечание 2:** **`!=`** это значит не равны. 
6. ### operators `! && ||` :
    - #### Описание:
        Бинарные операторы, кроме **`!`**, это унарный оператор.
        Эти операторы называют логическими.
        Работают только с **`bool`**.
    - #### Как работает:
        Это стандартные логические операции
        - **`&&`** это **И**
        - **`||`** это **ИЛИ**
        - **`!`** это **НЕ**
        
        [Подробнее.][5]

4. ### operators `~ & | ^ >> <<` :
    - #### Описание:
        Бинарные операторы, кроме `~`, это унарный оператор. Его схема выглядит так:
        ```
        ~(type) -> &type
        ```
        Эти операторы называют битовыми.
        Работают только с целыми типами.
    - #### Как работает:
        Числа в памяти хранятся в **двоичной системе исчисления**.
        
        > Например число `13 = 1101` в двоичной.
        
        Каждая цифра из двоичной записи числа хранится в одном **бите**. То есть **бит** может хранить либо `0`, либо `1`.
        
        **Байт** содержит `8` **бит**. 
        
        [Подробнее][1]

        > Например `unsigned int` занимает `4` **байта**. Это равно `32` **битам**.
        >  
        > ```c++
        > unsigned int a = 13;
        > // a = 0000 0000 0000 0000 0000 0000 0000 0000 1101 
        > // так a выглядит в памяти компьютера
        > ```
       
       **Перейдём к самим операторам:**

        - **`~`** все биты заменяет на обратные. То есть `0` переходит `1`, а `1` переходит в `0`.
            
            Пример:
            ```c++
            unsigned char c = 13;
            // c = 0000 1101
            c = ~c;
            // c = 1111 0010 = 242 в десятичной
            ```
        - **`&`** это бинарный оператор **И**, на вход даются 2 числа, например `а` и `b`, а в результате одно число, назавём его `c`.
            
            Пусть `a[n]` это `nый` **бит** числа.
            Тогда для любого `n` будет выполненно `c[n] = a[n] and b[n]`.

            >  **Замечание:** в С++ нет слова and, в данном месте оно приведенно для объяснения.
            
            Пример:
            ```c++
            unsigned char a = 13;
            unsigned char b = 20;
            // a = 0000 1101
            // b = 0001 0100
            unsigned char c = a & b;
            // c = 0000 0100
            ```
        - **`|`** это бинарный оператор **ИЛИ**, работает также как `&`, только в место **and** используется **or**.
        - **`^`** это бинарный оператор **XOR**, работает также как `&`, только в место **and** используется **xor**.
            [Подробнее.][4]
        - **`<<`** это бинарный оператор сдвига влево.
            Сдвигает все биты на заданное число влево (равносильно умножению на степень двойки). 
            
            Пример:

            ```c++
            unsigned char a = 13;
            // a = 0000 1101
            a = a << 3;
            // a = 0110 1000 = 104 в десятичной
            ```
        - **`>>`** это бинарный оператор сдвига вправо.
            Сдвигает все биты на заданное число вправо (равносильно делению на степень двойки). 
            
            Пример:

            ```c++
            unsigned char a = 105;
            // a = 0110 1001
            a = a >> 3;
            // a = 0000 1101 = 13 в десятичной
            ```
7. ### operators `+= /=` и т.д.:
    - #### Как работает:
        Объясним на примере оператора **`+=`**.
        Две следующих команды одинаковые:
        
        ```c++
        a = a + b; // 1
        a += b; // 2
        ```
        То есть мы к **`a`** прибавляем **`b`** и результат сразу записываем в **`a`**.
        Этот оператор позволяет более кратно и понятно писать код.
        Используйте его всегда когда есть возможность.

        С остальными предыдущими операторами работает так же:
        
        ```c++
        -=   *=   /=   &=   &&=   ы>>=   // и т.д.
        ```
8. ### operators `++ --` и т.д.:
     - #### Как работает:
        Эти операторы эквивалентны таким командам:

        ```c++
        a = a + 1; // добавляем 1 к a
        a++; // эквивален предыдущему
        ++a; // тоже эквивален предыдущему, но быстрее a++
        // в чем отличие a++ и ++a будет расказанно позже

        a = a - 1; // добавляем 1 к a
        a--; // эквивален предыдущему
        --a; // тоже эквивален предыдущему, но быстрее a--
        // в чем отличие a-- и --a будет расказанно позже
        ```
